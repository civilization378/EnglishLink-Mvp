# English Link · Hermes 项目规则

## 1. 项目身份

你现在是 English Link 微信小程序 MVP 的项目助手。

English Link 的核心目标是通过真实英文场景视频，帮助用户建立：

英文 → 场景 → 直接理解

的学习路径，而不是传统的“英文 → 中文翻译 → 理解”路径。

项目当前处于 MVP 阶段，最重要的是验证真实场景视频学习、答题反馈、学习记录、内容库和用户体验闭环，不追求复杂系统和过度功能。

---

## 2. 产品定位

English Link 不是普通背单词工具，也不是传统语法学习工具。

它的核心是：

- 去翻译化
- 真实英文视频场景
- 短视频沉浸式学习
- AI 语境解释
- 低门槛理解英文
- 通过真实场景建立英语语感

一句话定位：

不是教你翻译英文，而是让你在真实场景中直接理解英文。

目标用户包括：

- 初高中学生
- 英语基础薄弱但想提升理解能力的人
- 四六级或留学备考前期用户
- 想通过真实场景建立英语语感的人

## 4. 当前视觉方向

当前 UI 大方向已经确定，不要随意推翻。

### 首页方向

首页采用：

通用空间玻璃壁纸系统  
Universal Spatial Glass Wallpaper System

核心结构：

- 高质量场景壁纸
- 深色暗场 overlay
- 深色空间玻璃卡片
- 暖金色 CTA 按钮
- 高可读文字层级

首页不是视频播放页，而是：

- 学习控制台
- 今日推荐入口
- 学习进度展示
- 用户继续学习的入口

不要把首页改成普通白底卡片。  
不要做复杂假 tabBar。  
不要依赖单张固定壁纸或前景抠图。  
未来换壁纸时，只替换壁纸，不重做首页视觉系统。

### 学习页方向

学习页是短视频沉浸式学习页。

必须保持：

- 深色沉浸式
- 短视频感
- 高级克制
- 真实场景优先
- 信息层级清晰
- 不堆叠杂乱内容

禁止：

- 白底卡片式学习页
- 廉价大按钮
- 过多装饰
- 复杂无用功能
- 改变当前短视频学习逻辑

视频显示要求：

- 横屏 / 竖屏视频都必须完整显示
- 使用 contain，不裁切画面
- 切换视频时避免旧帧被新视频比例拉伸
- 切换动画要柔和，不要突兀黑屏

## 5. 当前阶段优先级

当前最重要的不是继续大幅打磨 UI，而是：

1. 保存稳定版本
2. 真机完整跑通流程
3. 补充更多真实视频内容
4. 建立稳定视频素材工作流
5. 小范围用户测试
6. 收集反馈
7. 再做 UI/UX 小修

当前要跑通的核心流程：

首页 → 学习页 → 观看视频 → 答题 → 抽屉内反馈 → 下一条 → 我的页

暂缓开发：

- streak 连续打卡系统
- 图鉴系统
- 今日完成页
- 错题重做
- 复杂成就系统
- 震动反馈
- 大型后端系统
- 复杂商业化功能

原则：

少做假功能，多做真实闭环。  
少做复杂系统，多做稳定体验。  
少做表面 UI，多补真实内容。  
先验证学习体验，再谈规模化。
