# English Link · 重要决策记录

## 产品方向

English Link 的核心方向是“真实英文场景视频 + 直接理解英文”，不是传统背单词或翻译式学习工具。

## UI 方向

首页采用“通用空间玻璃壁纸系统”。

学习页采用短视频沉浸式体验。

不要再改回白底卡片式，不要做普通学习卡片堆叠，不要做廉价按钮和复杂假功能。

## MVP 阶段原则

当前阶段不追求复杂功能，优先跑通真实学习闭环和内容生产流程。

重点是：

- 真实内容
- 稳定体验
- 小范围测试
- 用户反馈
- 可重复的视频素材工作流

暂时不做：

- streak
- 图鉴
- 复杂成就
- 大型后端
- 复杂商业化

## Hermes 使用原则

Hermes 是项目助手和自动化工作流助手，不直接替代 Cursor。

Cursor 更适合直接修改代码。

Hermes 更适合：

- 项目结构分析
- 文件检查
- 任务拆解
- 终端操作
- 视频素材工作流
- Skill 沉淀
- 长期项目记忆维护

## 模型选择

OpenRouter + Claude Sonnet 4.6 用于复杂代码和项目分析。

DeepSeek V4 Flash 用于便宜快速测试和中文规划。

DeepSeek V4 Pro 用于复杂推理和方案拆解。

## 云端部署决策

当前不急着部署 Hermes 到云端。

等本地视频工作流跑通 3–5 次、内容补充到约 10 条真实视频后，再考虑云端部署。

云端部署的目标应该是：

- 远程调用
- 24 小时在线
- 视频素材处理
- Telegram / Discord 私人助理
- 长期自动化工作流

不是为了单纯“看起来高级”。
